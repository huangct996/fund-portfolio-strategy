📖 回测逻辑说明

📅 报告期筛选规则

完整披露判断标准：
  • 权重总和 ≥ 50% 且 持仓数量 > 10只
  • 只有完整披露的报告期才会被纳入回测
部分披露处理：
  • 第一个报告期是部分披露 → 跳过，从下一个完整披露报告期开始建仓
  • 中间报告期是部分披露 → 使用上一期持仓（不调仓），继续持有
披露日期来源：使用Tushare API返回的实际公告日期（ann_date），而不是推测的标准披露日期
示例：
报告期 20190110 → 实际披露日 20190114（来自Tushare数据）
报告期 20190630 → 实际披露日 20190829（来自Tushare数据）

⏱️ 持有时间段计算

持有起始日：报告期的披露日期（在披露日的下一个交易日建仓）
持有结束日：
  • 有下一个有效报告期 → 下一个有效报告期的披露日期
  • 当前是最后一期 → 今天
示例：
报告期 20230630 → 持有起始日 20230829（实际披露日）
下一期 20230930 → 持有结束日 20231022（下一期实际披露日）
→ 持有时间段：2023-08-29 至 2023-10-22

📊 策略对比方式

持仓来源：两种策略使用完全相同的股票池（都来自该报告期披露的基金持仓）
权重差异：
策略类型 权重来源 计算方式
原策略 基金实际权重 使用 stk_mkv_ratio（占基金净资产比例）
权重归一化到100%
自定义策略 用户配置的策略 市值加权：按市值比例分配
综合得分：按归一化因子值加权
应用单只股票最大权重限制（默认10%）
期间收益率：持有起始日到持有结束日的单期收益率
累计收益率：从第一个完整披露报告期开始，使用复利公式累加
第一个报告期累计收益率 = 0%（建仓基点）
后续报告期累计收益率 = (1 + 上期累计收益率) × (1 + 本期期间收益率) - 1
超额收益 = 自定义策略累计收益率 - 原策略累计收益率

📋 部分披露处理

判断标准：权重总和 < 50%（一季报/三季报通常只披露前10-15只）
处理方式：
  • 使用上一个完整披露期的持仓
  • 不调仓，继续持有
  • 时间段仍从当前报告期的披露日开始

📈 累计收益率计算

选择多个报告期时，系统会分段计算每个报告期的收益率，然后复利累计：
累计收益率 = (1 + r₁) × (1 + r₂) × (1 + r₃) × ... - 1
示例：
第1期: +10% → 1.10
第2期: +5% → 1.05
第3期: -3% → 0.97
累计收益率 = 1.10 × 1.05 × 0.97 - 1 = 11.87%
