# 基金最大回撤显示问题诊断

## 问题现象
用户页面显示：基金最大回撤 = **0.87%**

## 后端验证
```bash
curl "http://localhost:3001/api/index-returns?..."
```

**结果**：
- 后端API返回：`maxDrawdown: 0.042380456422621754`
- 转换为百分比：`4.24%` ✅ **正确**

## 前端代码验证

### 显示逻辑（app.js 第1185行）
```javascript
<span class="risk-value" style="color: #FF6B6B;">
    ${(fundRisk.maxDrawdown * 100).toFixed(2)}%
</span>
```

**计算**：
- `fundRisk.maxDrawdown = 0.0424`
- `fundRisk.maxDrawdown * 100 = 4.24`
- `(fundRisk.maxDrawdown * 100).toFixed(2) = "4.24"`
- **应该显示：4.24%** ✅

## 问题根源分析

### 可能原因1：页面缓存
- 浏览器缓存了旧的API响应
- 旧数据中 `maxDrawdown = 0.0087`（0.87%）

### 可能原因2：参数不一致
- 页面实际发送的参数与测试参数不同
- 导致调用了不同的回测结果

### 可能原因3：前端代码未刷新
- 修改了 `app.js` 但浏览器未重新加载
- 仍在使用旧版本的JavaScript代码

## 解决方案

### 方案1：强制刷新浏览器
1. 按 `Ctrl + Shift + R`（Windows/Linux）或 `Cmd + Shift + R`（Mac）
2. 或在开发者工具中勾选 "Disable cache"
3. 重新点击"应用配置并计算"

### 方案2：清除浏览器缓存
1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方案3：使用测试页面验证
访问测试页面：
```
http://localhost:3001/test_api.html
```

该页面会：
1. 直接调用API
2. 显示原始返回数据
3. 计算并显示应该展示的值
4. 查找0.87%的可能来源

## 技术细节

### 后端计算逻辑
文件：`services/indexPortfolioService.js` 第1124-1137行

```javascript
// 最大回撤（基于净值数据）
let maxDrawdown = 0;
let peak = navData[0].accum_nav || navData[0].unit_nav;

for (let i = 1; i < navData.length; i++) {
  const nav = navData[i].accum_nav || navData[i].unit_nav;
  if (nav > peak) {
    peak = nav;
  }
  const drawdown = (peak - nav) / peak;
  if (drawdown > maxDrawdown) {
    maxDrawdown = drawdown;
  }
}
```

### 前端显示逻辑
文件：`public/app.js` 第1184-1186行

```javascript
<div class="risk-item">
    <span class="risk-label">最大回撤:</span>
    <span class="risk-value" style="color: #FF6B6B;">
        ${(fundRisk.maxDrawdown * 100).toFixed(2)}%
    </span>
</div>
```

**解决方法**：
1. 强制刷新浏览器（Ctrl+Shift+R 或 Cmd+Shift+R）
2. 或访问测试页面验证：http://localhost:3001/test_api.html
3. 确保看到的是最新的API响应数据
