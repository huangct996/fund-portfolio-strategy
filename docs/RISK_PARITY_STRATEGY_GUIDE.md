# 风险平价策略使用指南

## 📊 策略概述

风险平价策略是一种基于波动率的资产配置方法，旨在通过降低组合波动率来提高夏普比率。

### 核心原理

1. **风险平价加权**：权重与波动率成反比（权重 ∝ 1/波动率）
2. **EWMA波动率**：使用指数加权移动平均计算历史波动率，近期数据权重更高
3. **高频调仓**：支持季度或月度调仓，更及时地调整风险敞口
4. **交易成本**：可选择是否考虑交易成本，更真实地反映实际收益

### 预期效果

- **降低波动率**：通过风险平价加权，降低组合整体波动
- **提高夏普比率**：目标是达到指数策略的2倍左右
- **更稳定收益**：减少极端波动，提供更平滑的收益曲线

## 🚀 使用方法

### 1. 选择策略类型

在"策略配置"区域，选择 **"风险平价策略 ⭐"**

### 2. 配置参数

#### 波动率计算窗口（6-24个月）
- **推荐值**：12个月
- **说明**：用于计算历史波动率的时间窗口
- **调整建议**：
  - 较短窗口（6-9个月）：对市场变化更敏感，但可能过度反应
  - 较长窗口（18-24个月）：更稳定，但反应较慢

#### EWMA衰减系数（0.85-0.98）
- **推荐值**：0.94
- **说明**：控制近期数据的权重，值越大越重视历史数据
- **调整建议**：
  - 较小值（0.85-0.90）：更重视近期波动，适合快速变化的市场
  - 较大值（0.95-0.98）：更平滑，适合长期投资

#### 调仓频率
- **每年（跟随指数）**：与指数调仓同步，交易成本最低
- **每季度**：平衡收益和成本，**推荐用于初次测试**
- **每月**：最频繁，可能提升夏普比率，但交易成本较高

#### 交易成本
- **是否考虑**：建议勾选，更真实反映实际收益
- **成本比例**：默认0.30%（单边），可根据实际情况调整
- **说明**：包含买入和卖出的佣金、印花税等

#### 单只股票最大权重（5-30%）
- **推荐值**：15%
- **说明**：防止极端情况下权重过度集中

## 📈 测试建议

### 第一步：基础测试（推荐配置）

```
策略类型：风险平价策略
波动率窗口：12个月
EWMA衰减系数：0.94
调仓频率：每季度
考虑交易成本：是
交易成本比例：0.30%
最大权重：15%
```

**预期结果**：
- 夏普比率：1.5-2.0（目标是指数的2倍）
- 波动率：低于指数策略
- 最大回撤：小于或接近指数策略

### 第二步：对比测试

运行三次回测，对比不同调仓频率的效果：

1. **每年调仓**（基准）
   - 交易成本最低
   - 作为基准对比

2. **每季度调仓**（推荐）
   - 平衡收益和成本
   - 预期夏普比率提升20-30%

3. **每月调仓**（激进）
   - 最高频率
   - 观察交易成本对收益的影响

### 第三步：参数优化

根据测试结果，调整参数：

- **如果波动率仍然较高**：
  - 增加波动率窗口（如15-18个月）
  - 提高最大权重限制（如20%）

- **如果夏普比率未达标**：
  - 降低EWMA衰减系数（如0.90）
  - 提高调仓频率（改为每月）

- **如果交易成本过高**：
  - 降低调仓频率
  - 增加EWMA衰减系数（更平滑的权重变化）

## ⚠️ 注意事项

### 1. 数据缓存

首次运行风险平价策略时，系统需要获取大量历史日线数据：
- 每只股票需要12个月的日线数据（约240个交易日）
- 数据会自动缓存到本地数据库
- 后续运行会从数据库查询，速度更快

**首次运行时间**：
- 约50只股票 × 12个月 = 约5-10分钟
- 后续运行：< 1分钟

### 2. Tushare API限制

- 免费用户：每分钟120次请求
- 如果遇到限流，系统会自动等待
- 建议使用付费账户以获得更好的体验

### 3. 高频调仓的影响

**每季度调仓**（推荐）：
- 调仓次数：约32次（8年 × 4次/年）
- 交易成本：约0.3% × 32 = 9.6%
- 预期夏普比率提升：20-30%

**每月调仓**（激进）：
- 调仓次数：约96次（8年 × 12次/年）
- 交易成本：约0.3% × 96 = 28.8%
- 需要夏普比率提升 > 30% 才能覆盖成本

### 4. 市场环境影响

风险平价策略在不同市场环境下表现不同：

- **震荡市**：表现最佳，波动率降低效果明显
- **单边上涨市**：可能略逊于市值加权策略
- **熊市**：防御性更强，回撤控制更好

## 🛠️ 故障排查

### 问题1：夏普比率未达到预期

**可能原因**：
1. 波动率窗口过长，反应迟钝
2. 调仓频率过低
3. 交易成本设置过高

**解决方案**：
- 缩短波动率窗口至9个月
- 提高调仓频率至每季度或每月
- 检查交易成本设置是否合理

### 问题2：交易成本过高

**可能原因**：
1. 调仓频率过高
2. EWMA衰减系数过小，权重变化剧烈

**解决方案**：
- 降低调仓频率
- 增加EWMA衰减系数至0.96-0.98
- 适当提高最大权重限制

### 问题3：数据获取缓慢

**可能原因**：
1. 首次运行需要下载大量数据
2. Tushare API限流

**解决方案**：
- 耐心等待首次数据下载完成
- 后续运行会使用缓存数据，速度更快
- 考虑升级Tushare账户

### 问题4：某些股票波动率为0

**可能原因**：
1. 股票上市时间较短，历史数据不足
2. 停牌期间无交易数据

**解决方案**：
- 系统会自动使用默认波动率（2%日波动率）
- 这些股票权重会相对较高
- 可以通过最大权重限制来控制

## 📚 技术细节

### EWMA波动率计算公式

```
EWMA方差 = Σ(权重 × (收益率 - 均值)²) / Σ权重
权重 = λ^i，其中 i 从最近的数据开始递增
λ = EWMA衰减系数（如0.94）
```

### 风险平价权重计算

```
1. 计算每只股票的EWMA波动率 σᵢ
2. 计算逆波动率：wᵢ = 1/σᵢ
3. 归一化：Wᵢ = wᵢ / Σwⱼ
4. 应用最大权重限制
5. 重新归一化
```

### 交易成本计算

```
换手率 = Σ|新权重ᵢ - 旧权重ᵢ|
交易成本 = 换手率 × 成本比例
净收益率 = 毛收益率 - 交易成本
```
