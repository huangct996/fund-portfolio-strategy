# 自定义策略高收益率问题分析报告

## 执行摘要

**问题**：自定义策略累计收益率显示238.82%，远高于指数（73.72%）和基金（78.93%）

**根本原因**：自适应策略在某些调仓期将股票筛选得过于严格，导致持仓集中在1-2只股票，单只股票的异常收益被放大

**关键证据**：2021年7月调仓期，50只股票被筛选到只剩1只（002128.SZ），该股票3个月内上涨54.59%

---

## 详细分析

### 1. 问题发现过程

#### 初步怀疑
- 前端图表显示累计收益率241%
- 后端API返回238.82%
- 怀疑是累计收益率计算错误

#### 深入调查
- 检查累计收益率计算公式：**正确**
- 检查每日收益率数据：**正确**
- 检查调仓期收益率：**发现异常**

#### 关键发现
```
调仓期4（2021-07-01 至 2021-09-30）：
- 持仓股票数：1只（002128.SZ 深圳惠程）
- 持仓权重：100%
- 期间收益率：54.59%
- 累计收益率：从23.60% → 91.07%
```

### 2. 根本原因分析

#### 股票筛选过程（2021-07-01）

| 筛选步骤 | 条件 | 结果 |
|---------|------|------|
| 初始股票池 | 中证红利指数成分股 | 50只 |
| ROE筛选 | ROE >= 5% | 50只 → 2只 ❌ |
| 动量筛选 | 6月动量 >= -10% | 2只 → 2只 |
| 质量筛选 | 质量得分 >= 中位数 | 2只 → 1只 ❌ |

**问题**：
1. 自适应策略在市场温度较高时，自动提高了ROE要求到5%
2. 启用了质量得分筛选
3. 两轮严格筛选后，50只股票只剩1只

#### 为什么会出现这种情况？

**自适应策略逻辑**：
```javascript
// 市场温度较高时的防守策略
HOT: {
  maxWeight: 0.10,
  volatilityWindow: 12,
  minROE: 0.05,           // 提高ROE要求到5%
  maxDebtRatio: 0.8,
  filterByQuality: true,  // 启用质量筛选
  description: '谨慎防守 - 高估时降低仓位'
}
```

**问题**：
- ROE >= 5%的条件对于红利指数成分股过于严格
- 红利指数侧重高股息，不一定要求高ROE
- 质量筛选进一步减少了股票数量

## 解决方案

### 方案1：调整自适应策略的筛选参数（推荐）

**问题**：ROE >= 5%对红利指数过于严格

**建议**：
```javascript
HOT: {
  maxWeight: 0.10,
  volatilityWindow: 12,
  minROE: 0.02,           // 降低到2%，更适合红利股
  maxDebtRatio: 0.8,
  filterByQuality: true,
  description: '谨慎防守 - 高估时降低仓位'
}
```

### 方案2：设置最小持仓股票数

**问题**：持仓过于集中（1只股票）

**建议**：
```javascript
// 在股票筛选后添加检查
if (filteredStocks.length < 10) {
  console.warn(`⚠️  筛选后股票数过少（${filteredStocks.length}只），放宽筛选条件`);
  // 逐步放宽条件，直到至少有10只股票
}
```

### 方案3：添加持仓集中度限制

**问题**：单只股票权重过高（100%）

**建议**：
```javascript
// 在权重计算后添加检查
const maxSingleWeight = 0.20;  // 单只股票最大20%
Object.keys(weights).forEach(code => {
  weights[code] = Math.min(weights[code], maxSingleWeight);
});
// 重新归一化
```

### 方案4：关闭自适应策略的质量筛选

**问题**：质量筛选进一步减少股票数

**建议**：
```javascript
HOT: {
  maxWeight: 0.10,
  volatilityWindow: 12,
  minROE: 0.02,
  maxDebtRatio: 0.8,
  filterByQuality: false,  // 关闭质量筛选
  description: '谨慎防守 - 高估时降低仓位'
}
```
