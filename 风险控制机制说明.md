# 风险控制机制说明

## 问题
用户询问：
1. 报告中提到的"风险控制：通过波动率加权和最大权重限制"具体逻辑是怎么样？
2. 网页页面上怎么没有最大权重限制？

## 回答

### 1. 风险控制的具体逻辑

#### 波动率加权（风险平价核心）

**原理**：权重与波动率成反比，波动率越高的股票分配越低的权重。

**代码位置**：`services/indexPortfolioService.js` 第1648-1674行

**计算步骤**：

```javascript
// 1. 计算每只股票的波动率倒数
invVol = 1 / volatility

// 2. 归一化得到初始权重
weight = invVol / sum(所有股票的invVol)

// 3. 应用最大权重限制
weight = Math.min(weight, maxWeight)

// 4. 重新归一化，确保权重和为1
finalWeight = weight / sum(所有权重)
```

**示例**：
```
股票A：波动率 = 20% → invVol = 5.0 → 初始权重 = 25%
股票B：波动率 = 10% → invVol = 10.0 → 初始权重 = 50%
股票C：波动率 = 5%  → invVol = 20.0 → 初始权重 = 100%

应用10%最大权重限制后：
股票A：25% → 10% ✅
股票B：50% → 10% ✅
股票C：100% → 10% ✅

重新归一化（假设还有其他股票）：
总权重 = 30%，需要归一化到100%
```

#### 最大权重限制

**目的**：防止单只股票权重过高，降低集中度风险。

**代码位置**：`services/indexPortfolioService.js` 第1672行

```javascript
weight = Math.min(weight, maxWeight);
```

**默认值**：
- 风险平价策略：`maxWeight = 0.10` (10%)
- 自适应策略：根据市场状态动态调整（10%-20%）

**应用场景**：
1. **初始权重计算**：在计算风险平价权重时直接限制
2. **混合策略**：在混合市值加权时限制
3. **动量倾斜**：在应用动量因子后重新限制

**代码示例**（第1672行）：
```javascript
invVolatilities.forEach(s => {
  let weight = totalInvVol > 0 ? s.invVol / totalInvVol : 1 / stocks.length;
  weight = Math.min(weight, maxWeight);  // ✅ 应用最大权重限制
  riskParityWeights[s.tsCode] = weight;
});
```

#### 为什么风险平价策略没有配置项？

**原因1：策略设计**
- 风险平价策略的核心是"风险均衡"
- 10%是经过测试的合理上限
- 过高会破坏风险分散效果
- 过低会导致权重过于分散

**原因2：简化用户选择**
- 避免参数过多导致用户困惑
- 10%是业界常用的标准值
- 保持策略的纯粹性

**原因3：与自适应策略区分**
- 风险平价：固定参数，简单稳定
- 自适应风险平价：动态参数，灵活应变

### 3. 完整的风险控制流程

#### 风险平价策略的风险控制机制

```
1. 股票池筛选（可选）
   ↓
   剔除低质量股票（PE/PB/股息率）
   剔除弱势股票（动量<-10%）
   剔除高风险股票（ROE、负债率）

2. 波动率计算
   ↓
   使用EWMA方法计算历史波动率
   窗口：6个月（默认）

3. 风险平价权重
   ↓
   权重 ∝ 1/波动率
   波动率高 → 权重低
   波动率低 → 权重高

4. 最大权重限制
   ↓
   单只股票最大10%
   防止过度集中

5. 归一化
   ↓
   确保所有权重和为100%

6. 调仓执行
   ↓
   季度调仓（默认）
   降低交易成本
```

#### 风险控制的多层防护

| 层级 | 机制 | 目的 |
|-----|------|------|
| **第1层** | 股票池筛选 | 剔除低质量、弱势股票 |
| **第2层** | 波动率加权 | 降低高波动股票权重 |
| **第3层** | 最大权重限制 | 防止单只股票过度集中 |
| **第4层** | 季度调仓 | 降低交易成本和频繁调整 |

### 6. 总结

#### 风险控制逻辑

1. **波动率加权**：
   - 权重 = 1 / 波动率
   - 高波动 → 低权重
   - 低波动 → 高权重

2. **最大权重限制**：
   - 固定10%上限
   - 防止单只股票过度集中
   - 在多个环节应用

#### 网页配置现状

- **市值加权策略**：✅ 有配置项（5%-30%）
- **综合得分策略**：✅ 有配置项（5%-30%）
- **风险平价策略**：❌ 无配置项（硬编码10%）
- **自适应策略**：✅ 有说明（动态调整10%-20%）
