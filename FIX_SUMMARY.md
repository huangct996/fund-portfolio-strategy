# 修复总结报告

## 问题概述

**用户反馈**：
1. 自定义策略累计收益率显示241%，远高于指数（73.72%）和基金（78.93%）
2. 温度曲线图显示过多圆点，影响视觉效果
3. 需要查看每次调仓的详细情况（温度、调仓逻辑等）

---

## 问题根因

### 问题1：收益率过高（238.82%）

**根本原因**：自适应策略在某些调仓期将股票筛选得过于严格，导致持仓集中在极少数股票

**关键证据**：
- 2021年7月调仓期：50只股票 → 筛选后只剩1只（002128.SZ）
- 该股票3个月内上涨54.59%，导致累计收益率从23.60%跳升到91.07%
- 筛选过程：
  - ROE >= 5%：50只 → 2只
  - 质量得分筛选：2只 → 1只

**筛选过严的原因**：
- 自适应策略在市场温度较高时，自动提高ROE要求到5%
- 5%的ROE要求对红利指数成分股过于严格（红利股侧重高股息，不一定高ROE）
- 启用质量筛选进一步减少股票数量

### 问题2：温度曲线图圆点过多

**原因**：Chart.js配置中`pointRadius: 2`导致每个数据点都显示圆点

---

## 修复方案

### 修复1：调整自适应策略参数

**文件**：`services/marketThermometerService.js`

**修改内容**：
```javascript
// NORMAL级别
filterByQuality: false,  // 关闭质量筛选，避免持仓过度集中

// HOT级别（市场高估时）
minROE: 0.02,            // 从5%降低到2%，更适合红利股
filterByQuality: false,  // 关闭质量筛选
```

**效果**：
- 降低筛选严格度，保证足够的股票数量
- 避免持仓过度集中在少数股票

### 修复2：添加最小持仓股票数检查

**文件**：`services/stockFilterService.js`

**新增逻辑**：
```javascript
// 最小持仓股票数检查
const MIN_HOLDINGS = 10;
if (filteredStocks.length < MIN_HOLDINGS) {
  // 1. 先尝试关闭质量筛选
  // 2. 如果还不够，使用原始股票池
}
```

**效果**：
- 保证至少10只股票持仓
- 防止极端情况下持仓过度集中

### 修复3：温度曲线图去圆点

**文件**：`public/temperature.js`

**修改内容**：
```javascript
pointRadius: 0,              // 隐藏所有圆点
pointHoverRadius: 6,         // 鼠标悬停时显示
pointHitRadius: 15           // 增大感应区域
```

**效果**：
- 曲线平滑，无圆点
- 保留完整的交互功能
