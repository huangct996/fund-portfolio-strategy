# 缓存降级机制修复说明

## 修复时间
2026年1月13日 上午9:37

## 问题描述

用户反馈：当Tushare API调用失败并显示"⚠️ API调用失败，2秒后重试 (1/3)..."时，页面没有任何数据显示，即使数据库中有缓存数据。

## 问题根源

### 原有逻辑缺陷

在`services/tushareService.js`中，当API调用失败时，代码直接返回空数组或抛出错误，**没有尝试从数据库缓存中读取数据作为降级方案**。

#### 问题代码示例（修复前）

**1. 股票日线数据 (`getStockDailyWithCache`)**
```javascript

  return [];  // ❌ 直接返回空数组，没有尝试读取缓存
}
```

**2. 基金净值数据 (`getFundNavWithCache`)**

### 影响范围

当Tushare API出现以下情况时，页面会完全没有数据：
- ❌ API频率限制（每分钟700次）
- ❌ 网络超时
- ❌ API服务器故障
- ❌ Token失效
- ❌ 其他API错误

即使数据库中有完整的历史缓存数据，也无法使用。

## 修复方案

### 核心思路：三层降级策略

```
1. 优先使用数据库缓存（如果已完整同步）
   ↓ 失败
2. 调用Tushare API获取最新数据
   ↓ 失败
3. 降级到数据库缓存（即使数据可能不完整）
```

### 修复内容

#### 1. 股票日线数据 (`getStockDailyWithCache`)

**文件**: `services/tushareService.js` 第1195-1230行

**修复后**:
```javascript
} catch (error) {
  console.error(`获取股票日线数据失败 (${tsCode}):`, error.message);
  
  // API调用失败，尝试从数据库读取缓存数据作为降级方案
  try {
    console.log(`⚠️  尝试从数据库缓存读取 ${tsCode} 的数据...`);
    const cachedDailyData = await dbService.getStockDaily(tsCode, startDate, endDate);
    const cachedAdjFactorData = await dbService.getAdjFactor(tsCode, startDate, endDate);
    
    if (cachedDailyData.length > 0) {
      console.log(`✅ 从缓存读取到 ${cachedDailyData.length} 条数据`);
      
      // 如果有复权因子，计算后复权价格
      if (cachedAdjFactorData.length > 0) {
        const adjFactorMap = {};
        cachedAdjFactorData.forEach(item => {
          adjFactorMap[item.trade_date] = item.adj_factor;
        });
        
        const latestAdjFactor = cachedAdjFactorData[cachedAdjFactorData.length - 1].adj_factor;
        return cachedDailyData.map(item => ({
          ...item,
          adj_close: item.close * (latestAdjFactor / (adjFactorMap[item.trade_date] || 1))
        }));
      }
      
      return cachedDailyData;
    } else {
      console.log(`❌ 缓存中没有 ${tsCode} 的数据`);
    }
  } catch (cacheError) {
    console.error(`读取缓存数据失败:`, cacheError.message);
  }
  
  return [];
}
```

## 数据新鲜度说明

### 缓存数据的时效性

使用缓存数据时，数据可能不是最新的：

| 数据类型 | 更新频率 | 缓存时效 |
|---------|---------|---------|
| 股票日线 | 每日收盘后 | 最多延迟1天 |
| 基金净值 | 每日收盘后 | 最多延迟1天 |
| 指数数据 | 每日收盘后 | 最多延迟1天 |
| 财务数据 | 季度更新 | 最多延迟1季度 |

### 缓存数据提示

建议在前端添加提示，告知用户正在使用缓存数据：

```javascript
// 前端显示示例
if (dataFromCache) {
  showWarning('⚠️ 当前使用缓存数据，可能不是最新数据');
}
```

## 注意事项

### 1. 缓存数据完整性

缓存降级只在数据库中有数据时才有效。如果是首次访问且API失败，仍然会显示空白。

### 2. 数据一致性

使用缓存数据时，不同数据源的时间可能不一致：
- 股票数据：2026-01-10
- 基金数据：2026-01-09（可能延迟1天）

### 3. 性能影响

降级到缓存时，会额外进行一次数据库查询，但性能影响很小（通常<100ms）。
