## 问题描述

用户反馈两个问题：
1. **图表横线问题**：累计收益率图表中基金数据（512890.SH）有很多横线
2. **最大回撤显示错误**：基金最大回撤显示为0.87%，明显不合理

## 问题分析

### 1. 图表横线问题

**原因**：
- 基金净值数据在周末和节假日没有更新
- 使用 `plot()` 绘制时，matplotlib会自动连接所有数据点
- 当数据点之间有较大时间间隔时，会出现横线

**数据验证**：
```python
# 基金数据点数: 1217个
# 策略数据点数: 1217个（每个交易日都有）
# 基金数据有缺失，导致横线出现
```

### 2. 最大回撤计算错误

**后端计算结果**（正确）：
```
自定义策略最大回撤: 26.09%
指数最大回撤: 17.54%
基金最大回撤: 4.24%
```

**报告显示结果**（错误）：
```
基金最大回撤: 0.87%  ❌
```

**原因**：
报告生成脚本中的回撤计算公式错误：

```python
# 错误的公式
dd = (val - peak) / (1 + peak) * 100
```

这个公式的问题：
1. 分母使用 `(1 + peak)` 不正确
2. 应该使用 `(peak - val) / peak` 计算回撤百分比

### 修复2：最大回撤计算错误

**文件**：`generate_report.py` 第229-237行

**修改前**：
```python
# 计算最大回撤
peak = cumulative[0]
drawdowns = []
for val in cumulative:
    if val > peak:
        peak = val
    dd = (val - peak) / (1 + peak) * 100  # ❌ 错误公式
    drawdowns.append(dd)
```

**修改后**：
```python
# 计算最大回撤（正确的公式）
peak = 1.0  # 初始净值为1
drawdowns = []
for val in cumulative:
    current_nav = 1 + val  # 当前净值
    if current_nav > peak:
        peak = current_nav
    dd = (peak - current_nav) / peak * 100  # 回撤 = (峰值 - 当前值) / 峰值
    drawdowns.append(-dd)  # 负值表示回撤
```

**关键改动**：
1. **初始峰值**：从 `cumulative[0]` 改为 `1.0`（初始净值）
2. **当前净值**：`current_nav = 1 + val`（累计收益率转换为净值）
3. **回撤公式**：`(peak - current_nav) / peak * 100`
4. **负值表示**：`-dd`（回撤用负值表示下跌）

### 回撤计算公式说明

**正确的最大回撤计算方法**：

```
回撤 = (峰值净值 - 当前净值) / 峰值净值 × 100%
```

**示例**：
- 峰值净值：1.50（累计收益50%）
- 当前净值：1.20（累计收益20%）
- 回撤 = (1.50 - 1.20) / 1.50 × 100% = 20%

**错误公式的问题**：
```python
dd = (val - peak) / (1 + peak) * 100
```
- `val` 是累计收益率（如0.5表示50%收益）
- `peak` 也是累计收益率
- 分母 `(1 + peak)` 不是正确的峰值净值
- 分子 `(val - peak)` 在回撤时是负数，但公式没有取绝对值

## 技术细节

### Matplotlib绘图方法对比

**plot() 方法**：
- 连接所有数据点形成连续曲线
- 适合连续的时间序列数据
- 数据缺失时会出现横线

**scatter() 方法**：
- 绘制独立的散点
- 不连接数据点
- 数据缺失时不会出现横线
- 适合有缺失的数据

### 回撤计算的数学原理

**净值曲线**：
```
净值 = 1 + 累计收益率
```

**回撤定义**：
```
回撤 = (历史最高净值 - 当前净值) / 历史最高净值
```

**最大回撤**：
```
最大回撤 = max(回撤序列)
```

**示例计算**：
```python
# 累计收益率序列
cumulative = [0.0, 0.1, 0.2, 0.15, 0.25, 0.20]

# 净值序列
nav = [1.0, 1.1, 1.2, 1.15, 1.25, 1.20]

# 峰值序列
peak = [1.0, 1.1, 1.2, 1.2, 1.25, 1.25]

# 回撤序列
drawdown = [0%, 0%, 0%, 4.17%, 0%, 4.0%]

# 最大回撤
max_drawdown = 4.17%
```
