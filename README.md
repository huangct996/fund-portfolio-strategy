# 基金复制策略系统

## 项目概述

本系统是一个基于指数成分股的基金复制和策略回测平台，通过复制中证红利低波动指数（h30269.CSI）的成分股，实现对512890.SH基金的跟踪和优化。系统支持多种策略类型，包括市值加权、综合得分和风险平价策略。

## 核心功能

### 1. 数据管理

**统一数据访问模式**
- **本地优先**：首先查询本地MySQL数据库
- **API补充**：本地无数据时自动调用Tushare API获取
- **智能缓存**：对超过一个月的缺失数据打标记，避免重复查询
- **数据类型**：
  - 基金净值数据（fund_nav）
  - 指数日线数据（index_daily）
  - 股票日线数据（stock_daily）
  - 股票基本信息（stock_basic_info）
  - 指数成分股权重（index_weight）

### 2. 策略类型

#### 2.1 市值加权策略（默认）
- 按照成分股市值比例分配权重
- 单只股票最大权重限制（默认10%）
- 超限权重按比例分配给其他股票

#### 2.2 综合得分策略
- 市值因子（默认权重50%）
- 股息率因子（默认权重30%）
- 质量因子（默认权重20%）
  - PE倒数
  - PB倒数
  - ROE
- 根据综合得分分配权重

#### 2.3 风险平价策略
- 使每只股票对组合总风险的贡献相等
- 使用EWMA计算历史波动率
- 支持协方差矩阵调整
- 支持质量因子倾斜
- 支持动量因子倾斜
- 支持股票池筛选（ROE、负债率、动量）

**调仓频率**
- 年度调仓：跟随指数年度调整
- 季度调仓：每季度调仓一次
- 月度调仓：每月调仓一次

### 3. 风险指标

系统计算并展示以下风险指标：

- **累计收益率**：整个回测期间的总收益率
- **年化收益率**：按年化计算的收益率
- **年化波动率**：收益率的标准差（年化）
- **最大回撤**：从历史最高点到最低点的最大跌幅
- **夏普比率**：超额收益与波动率的比值
- **索提诺比率**：超额收益与下行波动率的比值
- **卡玛比率**：年化收益率与最大回撤的比值
- **跟踪误差**：自定义策略相对指数策略的收益率偏离

### 4. 可视化展示

#### 4.1 累计收益率曲线
- 自定义策略收益率曲线（蓝色实线）
- h30269.CSI指数收益率曲线（橙色实线）
- 512890.SH基金收益率曲线（红色虚线）
- 调仓日期标记点
- 使用线性插值平滑曲线

#### 4.2 风险指标对比
- 三列展示：自定义策略、指数策略、512890.SH基金
- 6项核心指标对比
- 风险收益指标对比表

#### 4.3 持仓明细
- 三列展示：
  - h30269.CSI指数持仓（原始权重）
  - 512890.SH基金持仓（按季度披露）
  - 自定义策略持仓（调整后权重）
- 支持按调仓期切换查看
- 显示调仓变化（新增/移除股票）

## 安装和使用

### 环境要求
- Node.js >= 14.x
- MySQL >= 5.7
- Tushare API Token

### 安装步骤

1. 克隆项目
```bash
git clone <repository-url>
cd fund_replication
```

2. 安装依赖
```bash
npm install
```

3. 配置环境变量
创建 `.env` 文件：
```env
TUSHARE_TOKEN=your_tushare_token
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=fund_replication
```

4. 启动服务
```bash
node server.js
```

5. 访问系统
打开浏览器访问：http://localhost:3001

## 使用说明

### 基本操作流程

1. **配置策略参数**
   - 选择策略类型（市值加权/综合得分/风险平价）
   - 设置回测日期范围
   - 调整策略参数

2. **执行回测**
   - 点击"应用配置并计算"按钮
   - 系统自动获取数据并计算
   - 展示收益率曲线和风险指标

3. **查看结果**
   - 分析累计收益率曲线
   - 对比风险指标
   - 查看持仓明细
   - 切换调仓期查看历史持仓

### 高级功能

#### 风险平价策略配置
- **波动率窗口**：计算历史波动率的月数（默认12个月）
- **EWMA衰减系数**：指数加权移动平均的衰减系数（默认0.94）
- **调仓频率**：yearly/quarterly/monthly
- **交易成本**：是否考虑交易成本（默认0.1%）
- **质量因子倾斜**：是否使用质量因子调整权重
- **协方差调整**：是否使用协方差矩阵优化
- **动量因子倾斜**：是否使用动量因子调整权重
- **股票池筛选**：ROE、负债率、动量筛选条件

#### 指数成分股查询
- 按调仓日期查询指数成分股
- 查看调仓变化（新增/移除股票）
- 导出成分股列表

## 数据流程

```
用户请求
    ↓
检查本地数据库
    ↓
    ├─ 有数据 → 返回数据
    └─ 无数据 → 检查缺失标记
                  ↓
                  ├─ 已标记 → 跳过查询
                  └─ 未标记 → 调用Tushare API
                              ↓
                              ├─ 成功 → 保存到数据库 → 返回数据
                              └─ 失败/空数据 → 标记缺失（如果超过1个月）
```

## 性能优化

1. **数据缓存**
   - 本地数据库缓存所有历史数据
   - 缺失数据标记避免重复API调用
   - 批量获取数据减少API调用次数

2. **计算优化**
   - 使用每日数据计算精确风险指标
   - 前复权价格计算真实收益率
   - 批量处理股票数据

3. **前端优化**
   - 线性插值平滑曲线
   - 响应式布局适配不同屏幕
   - 表格固定布局防止变形

## 注意事项

1. **数据准确性**
   - 首次运行需要从Tushare获取大量历史数据，耗时较长
   - 基金持仓数据按季度披露，存在时间延迟
   - 停牌股票可能导致部分日期无数据

2. **API限制**
   - Tushare API有调用频率限制（700次/分钟）
   - 系统已实现自动限流和重试机制
   - 建议避免频繁重复查询相同数据

3. **回测局限**
   - 回测结果不代表未来表现
   - 未考虑滑点、冲击成本等实际交易成本
   - 历史数据可能存在幸存者偏差

## 常见问题

### Q1: 首次运行很慢怎么办？
A: 首次运行需要从Tushare获取大量历史数据，建议耐心等待。后续运行会直接从本地数据库读取，速度会快很多。

### Q2: 为什么基金曲线有些地方是水平的？
A: 基金净值数据在周末和节假日不更新，系统使用线性插值平滑曲线。如果曲线末尾水平，说明基金净值数据还未更新到最新日期。

### Q3: 如何清空数据库重新开始？
A: 删除MySQL数据库中的所有表，重启服务器会自动重建表结构。

### Q4: 支持哪些指数和基金？
A: 当前版本支持h30269.CSI指数和512890.SH基金。如需支持其他指数和基金，需要修改配置文件。
