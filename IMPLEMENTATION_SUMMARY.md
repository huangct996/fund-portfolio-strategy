### 1. 数据分析与调仓日期识别

#### 创建分析脚本 (`analyze_index_changes.js`)
- 分析数据库中的指数成分股数据
- 识别成分股发生变化的日期
- **分析结果**：
  - 总交易日数：120个
  - 真正的调仓次数：11次
  - 调仓日期列表：
    ```
    20151231, 20161230, 20171229, 20181228, 20190131, 
    20191231, 20201231, 20211231, 20221230, 20231229, 20241231
    ```
  - 符合指数编制方案（每年12月调整一次）

#### 核心发现
- **问题根源**：Tushare API返回的`index_weight`表包含每个交易日的权重数据
- **权重变化原因**：即使成分股不变，由于股价波动，各成分股的市值占比每天都在变化
- **解决方案**：通过对比相邻交易日的成分股列表，只识别成分股发生变化的日期

## 🔧 技术实现细节

### 调仓日期识别算法
```javascript
// 对比相邻日期的成分股列表
for (let i = 0; i < allDates.length; i++) {
  const currentStocks = new Set(/* 当前日期的成分股 */);
  
  if (i === 0) {
    // 第一个日期，记录为初始调仓
    rebalanceDates.push(date);
  } else {
    // 检查成分股是否发生变化
    const added = [...currentStocks].filter(code => !prevStocks.has(code));
    const removed = [...prevStocks].filter(code => !currentStocks.has(code));
    
    if (added.length > 0 || removed.length > 0) {
      // 成分股发生变化，记录为调仓日期
      rebalanceDates.push(date);
    }
  }
  
  prevStocks = currentStocks;
}
```

### 调仓变化数据结构
```javascript
{
  date: "20231229",           // 调仓日期
  prevDate: "20221230",       // 上一期日期
  totalStocks: 50,            // 当前成分股数量
  prevTotalStocks: 50,        // 上一期成分股数量
  added: ["600016.SH", ...],  // 新增的成分股
  removed: ["000401.SZ", ...],// 移除的成分股
  addedCount: 20,             // 新增数量
  removedCount: 20,           // 移除数量
  isInitial: false            // 是否为初始调仓
}
```
