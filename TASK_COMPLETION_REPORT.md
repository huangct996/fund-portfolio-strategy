## 🔍 技术实现细节

### 核心修改点

1. **dbService.getIndexWeightDates()**
   ```javascript
   // 对比相邻日期的成分股列表
   for (let i = 0; i < allDates.length; i++) {
     const currentStocks = new Set(/* 当前日期的成分股 */);
     
     if (i === 0) {
       rebalanceDates.push(date); // 第一个日期
     } else {
       const added = [...currentStocks].filter(code => !prevStocks.has(code));
       const removed = [...prevStocks].filter(code => !currentStocks.has(code));
       
       if (added.length > 0 || removed.length > 0) {
         rebalanceDates.push(date); // 成分股发生变化
       }
     }
   }
   ```

2. **数据流向**
   ```
   Tushare API (120个trade_date)
   ↓
   dbService.getIndexWeightDates() (识别真正的调仓)
   ↓
   返回11个真正的调仓日期
   ↓
   前端/后端使用这11个日期进行回测和查询
   ```

### 关键API接口

| 接口 | 用途 | 返回数据 |
|------|------|----------|
| `/api/rebalance-dates` | 获取所有真正的调仓日期 | 11个调仓日期 |
| `/api/rebalance-changes` | 获取调仓变化详情 | 每次调仓的新增/移除股票 |
| `/api/index-returns` | 计算回测收益率 | 使用真正的调仓日期计算 |
| `/api/index-constituents` | 查询成分股 | 指定调仓日期的成分股列表 |
